1. APIMarket deve permettere di registrare le api di un microservizio attraverso la registrazione e pubblicazione della sua interfaccia (obbl)
	1.1 ApiMarket deve permettere la creazione di profili utente univoci tramite cui registrare le proprie API e acquistare ApiKey (obbl)
	    1.1.1 La registrazione di un nuovo utente deve chiedere le sue generalità. (Des)
	        1.1.1.1 La registrazione di un nuovo utente deve richiedere il suo Nome e Cognome. (Fac)
	            1.1.1.1.1 Il nome e il cognome devono essere composto da un minimo di 3 caratteri ad un massimo di 30 alfabetici. (Fac)
	            1.1.1.1.2 il nome e il cognome devono essere composto da soli caratteri alfabetici. (Fac)
	        1.1.1.2 La registrazione di un nuovo utente deve richiedere il suo numero di telefono. (Fac)
	            1.1.1.2.1 Il numero di telefono deve essere composto da sole cifre. (Fac)
	            1.1.1.2.2 Il numero di telefono deve essere formato da un minimo di 6 cifre ad un massimo di 30. (Fac)
	        1.1.1.3 La registrazione di un nuovo utente deve richiedere il suo paese di provenienza. (Des)
	            1.1.1.3.1 Il paese di provenienza deve corrispondere ad uno Stato esistente. (Des)
	    1.1.2 La registrazione di un nuovo utente deve richiedere il suo indirizzo email. (Obbl)
	        1.1.2.1 L'email deve avere un formato valido del tipo xxx@yyy.zz (obbl)
	        1.1.2.2 L'email deve essere univoca all'interno del sistema. (obbl)
	    1.1.3 La registrazione di un nuovo utente deve richiedere una password. (obbl)
	        1.1.3.1 La password deve essere composta almeno da 6 caratteri e al massimo da 20 caratteri. (des)
	        1.1.3.2 La password deve essere composta di caratteri alfanumerici e deve contenere almeno un carattere speciale. (fac)
	        1.1.3.3 La password deve essere ripetuta per controllare la sua correttezza. (fac)
	    1.1.4 La registrazione di un nuovo utente deve richiedere lo username. (obbl)
	        1.1.4.1 Lo username deve essere composto di caratteri alfanumerici, inoltre deve essere composto da un minimo di 4 caratteri ad un massimo di 30. (fac)
	        1.1.4.2 Lo username deve essere unico all'interno del sistema. (obbl)
	   1.1.5 APIMarket deve controllare i dati inseriti in fase di registrazione e deve fornire un messaggio di errore nel non siano conformi. (des)
	        1.1.5.1 Il sistema deve visualizzare un errore di inserimento Nome durante la registrazione. (des)
	        1.1.5.2 Il sistema deve visualizzare un errore di inserimento Cognome durante la registrazione. (des)
	        1.1.5.3 Il sistema deve visualizzare un errore di inserimento Numero di telefono durante la registrazione. (des)
	        1.1.5.4 Il sistema deve visualizzare un errore di inserimento email durante la registrazione. (des)
	        1.1.5.5 Il sistema deve visualizzare un errore di inserimento password durante la registrazione. (des)
	        1.1.5.6 Il sistema deve visualizzare un errore di inserimento di errore conferma password durante la registrazione. (des)
		1.1.6 APIMarket deve controllare i dati inseriti durante la registrazione di un nuovo utente, e fornire un messaggio di errore
		in caso di dati già esistenti.(desid)
		1.1.7 APIMarket deve mostrare un messaggio di conferma per l'avvenuta registrazione. (desid)
		    1.1.7.1 APIMarket deve mostrare un messaggio che inviti l'utente a controllare la sua casella email per verificare la presenza di un link di verifica. (fac)
		1.1.8 APIMarket deve terminare la registrazione utente solo una volta che l'utente ha premuto sul link di conferma inviato nella sua email. (fac)
		    1.1.8.1 APIMarket deve fornire la pssibilità di reinviare la mail di conferma. (fac)
		    1.1.8.2 APIMarket deve fornire la possibilità di modificare l'indirizzo email al quale inviare la mail di conferma. (fac)
	1.2 APIMarket deve permettere l'autenticazione di un utente registrato. (obbl)
	    1.2.1 L'autenticazione di un utente registrato deve richiedere il nome utente o in alternativa la sua email. (obbl)
	    1.2.2 L'autenticazione di un utente registrato deve richiedere la sua password. (obbl)
	    1.2.3 APIMarket deve mostrare un errore all'utente nel caso l'autenticazione non vada a buon fine. (des)
	        1.2.3.1 APIMarket deve mostrare un errore e bloccare l'autenticazione nel caso il nome utente o la sua email siano errati.
	        1.2.3.2 APIMarket deve mostrare un errore e bloccare l'autenticazione nel c
	    1.1.8 ApiMarket deve fornire una funzionalità per il recupero delle password nel caso questa venga smarrita dall'utente. (facolt)
		1.1.3 Apimarket deve fornire all'utente registrato una pagina in cui modificare i proprio dati inseriti al momento della registrazione. (facolt)
		1.1.4 ApiMarket deve fornire delle funzionalità di monitoramento da parte dell'utente delle sue API, per quanto concerne vendite, guadagni e statistiche di utilizzo. (desid)
	1.2 APIMarket deve fornire un sistema di notifiche che permetta all'utente fornitore dell'API di avvisare i fruitori della sua API, e permetta di comunicare loro gli aggiornamenti. (desider)
	1.3 APIMarket deve permettere di impostare un nome per la sua API al momento del caricamento della sua interfaccia. (obbl)
		1.3.1 APIMarket deve permettere all'utente registrato che ha caricato un'API di modificare il nome della sua API (facolt)
	1.4 APIMarket deve permettere di impostare un prezzo per la sua API al momento del caricamento della sua interfaccia. (obbl)
		1.4.1 APIMarket deve permettere all'utente registrato che ha caricato un'API di modificare il prezzo della sua API (desider)
	1.5 APIMarket deve permettere di impostare i dipi di policy di vendita con cui vuole rendere disponibile la sua API al momento del caricamento della sua interfaccia. (desider)
		1.5.1 APIMarket deve permettere all'utente registrato che ha caricato un'API di modificare il i tipi di policy di vendita della sua API (facolt)
	1.6 APIMarket deve permettere di impostare un testo introduttivo che descriva la sua API al momento del caricamento della sua interfaccia. (obbl)
		1.6.1 APIMarket deve permettere all'utente registrato che ha caricato un'API di modificare il il testo introduttivo che descrive la sua API. (obbl)
2.ApiMarket deve permettere di consultare le API di un microservizio e la documentazione relativa. (obbl)
	?2.1 APIMarket deve permettere all'utente di caricare la documentazione assieme alla propria API, e deve assicurarsi che lo faccia. (obbl)
	2.2 ApiMarket deve permettere la modifica della documentazione da parte dell'utente proprietario dell'API. (obbl)
	2.3 APIMarket deve permettere a tutti gli utenti la ricerca delle API caricate nel sito dagli utenti registrati. (obbl)
		2.3.1 APIMarket deve permetter di eseguire ricerche filtrate e ordinate per categoria, prezzo, prestazioni... (facolt)
3.APIMarket deve associare una APIKey d'uso per ogni API registrata e per ogni utente che decide di acquistarla. (obbl)
	3.1 Ciascuna APIKey deve essere unica e può essere usata dall'utente che l'ha acquistata per accedere alla API a cui è collegata. (obbl)
		3.1.1 Se la policy di acquisto di una APIKey termina e non viene rinnovata la APIKey cessa di funzionare e non permette più l'utilizzo dell'API a cui è collegata. (obbl)
		3.1.2 Ogni APIKey permette uno scambio di dati giornaliero limitato. Se la quantità di dati scambiati supera la soglia la APIKey cessa di funcionare fino al giorno successivo. (facolt)
	3.2 Se la APIKey è contraffatta o non valida allora la chiamata all'API viene bloccata dall'APIMaket. (obbl)
	3.3 Ciascun utente registrato può possedere più APIKey, e può monitorare le prestazioni, la durata della licenza ed eventuali aggiornamenti dell'API nella sua pagina utente per mezzo delle APIKey.(desider)
		?3.3.1 L'utente riceve dall'APIMarket un avviso una settimana prima della scadenza di una APIKey da lui acquistata, o prima dell'esaurimento di traffico acquistato. (facolt)
4.ApiMarket deve fornire funzionalità di monitoramento delle API da parte di utenti esterni in base alle APIKey rilasciate. (obbl)
	4.1 APIMarket deve fornire per ogni API un servizio che monitori il numero di crash dell'API tra tutte le APIkey rilasciate. (obbl)
	4.2 APIMarket deve fornire statistiche su numeri di volte che una API viene utilizzata tramite APIKey (obbl)
	4.3 APIMerket deve fornure statistiche riguardo alla qantità di traffico media scambiata dall'API tramite APIKey (desider).
5. APIMarket deve permettere ad ogni utente loggato che abbia registrato l'interfaccia di un'API, di poterla rimuovere da APIMarket. (obbl)
6. APIMarket deve fornire al committente del capitolato un account Admin con poteri di Super-User. (obbl)
    6.1 Un utente Admin deve poter eseguire qualuque funzione che può eseguire un Utente registrato. (obbl)
    6.2 Un utente Admin deve avere la possibilità di modificare i dati anagrafici di qualsiasi utente. (facolt)
    6.3 Un utente Admin deve poter eliminare un commento di qualsiasi utente (facolt)
    6.4 Un utente Admin deve poter eliminare qualsiasi utente. (obbl)
    6.5 Un utente Admin deve poter modificare i dati di una API la cui interfaccia è stata caricata su APIMarket. (obbl)
    6.6 Un utente Admin deve avere la possibilità di modificare la documentazione relativa ad una API sull'APIMarket. (obbl)
    6.7 Un utente Admin deve poter elimindare una API da APIMarket (obbl)
    6.8 Un utente Admin deve poter ricercare qualunque utente tramite una barra di ricerca dedicata (facolt)
