1. ApiMarket deve permettere la creazione di profili utente univoci tramite cui registrare le proprie API e acquistare ApiKey (obbl)
	1.1 La registrazione di un nuovo utente deve chiedere le sue generalità. (Des)
		1.1.1 La registrazione di un nuovo utente deve richiedere il suo Nome e Cognome. (Fac)
		1.1.2 La registrazione di un nuovo utente deve richiedere il suo numero di telefono. (Fac)
		1.1.3 La registrazione di un nuovo utente deve richiedere il suo paese di provenienza. (Des)
	1.2 La registrazione di un nuovo utente deve richiedere il suo indirizzo email. (Obbl)
		1.2.1 L'email deve essere univoca all'interno del sistema. (obbl)
	1.3 La registrazione di un nuovo utente deve richiedere una password. (obbl)
		1.3.1 La password deve essere ripetuta per controllare la sua correttezza. (fac)
	1.4 La registrazione di un nuovo utente deve richiedere lo username. (obbl)
		1.4.1 Lo username deve essere unico all'interno del sistema. (obbl)
	1.5 La registrazione di un nuovo utente deve permettere all'utente di registrare una carta di credito nel sistema.
		1.5.1 La registrazione della carta di credito nel sistema deve essere opzionale.
		1.5.2 La registrazione della carta di credito deve richiedere il numero di carta di credito.
		1.5.3 La registrazione della carta di credito deve richiedere il nome dell'intestatario.
		1.5.4 La registrazione della carta di credito deve richiedere la data di scadenza.
		1.5.5 La registrazione della carta di credito deve richiedere il codice CVV2.
	1.6 APIMarket deve controllare i dati inseriti in fase di registrazione e deve fornire un messaggio di errore nel caso non siano conformi. (des)
	1.7 APIMarket deve controllare i dati inseriti durante la registrazione di un nuovo utente, e fornire un messaggio di errore
	in caso di dati già esistenti.(desid)
	1.8 APIMarket deve mostrare un messaggio di conferma per l'avvenuta registrazione. (desid)
		1.8.1 APIMarket deve mostrare un messaggio che inviti l'utente a controllare la sua casella email per verificare la presenza di un link di verifica. (fac)
	1.9 APIMarket deve terminare la registrazione utente solo una volta che l'utente abbia visitato il link di conferma inviato nella sua email. (fac)
		1.9.1 APIMarket deve fornire la possibilità di reinviare la mail di conferma. (fac)
		1.9.2 APIMarket deve fornire la possibilità di modificare l'indirizzo email al quale inviare la mail di conferma. (fac)
2 APIMarket deve permettere l'autenticazione di un utente registrato. (obbl)
	2.1 L'autenticazione di un utente registrato deve richiedere il nome utente o in alternativa la sua email. (obbl)
	2.2 L'autenticazione di un utente registrato deve richiedere la sua password. (obbl)
	2.3 APIMarket deve mostrare un errore all'utente nel caso l'autenticazione non vada a buon fine. (des)
	2.4 APIMarket deve rendere disponibile una funzionalità per il recupero della password qualora l'utente dimentichi la sua. (des)
		2.4.1 APIMarket, nella procedura per il recupero password, deve inviare una password sostitutiva alla mail dell'utente che ne ha fatto richiesta e deve notificarlo all'utente. (fac)
		2.4.2 APIMarket deve permettere all'utente di accedere al suo account tramite la password sostitutiva che gli ha fornito via email. (fac)
3. APIMarket deve permettere di registrare le API di un microservizio attraverso la registrazione e pubblicazione della sua interfaccia (obbl)
	3.1 APIMarket deve fornire una funzionalità nella pagina utente per pubblicare l'interfaccia di un suo microservizio. (obbl)
	3.2 APIMarket deve permettere di impostare un nome unico per il suo microservizio al momento del caricamento della sua interfaccia. (obbl)
		3.1.1 APIMarket deve mostrare un errore all'utente qualora tenti di impostare un nome già esistente in APIMarket per il suo microservizio.
	3.3 APIMarket deve permettere di impostare un prezzo per il suo microservizio al momento del caricamento della sua interfaccia. (obbl)
		3.3.1 APIMarket deve far registrare una carta di credito all'utente qualora tenti di impostare un prezzo per la sua API e non abbia una carta di credito registrata nel suo account.
	3.4 APIMarket deve permettere di impostare i tipi di policy di vendita con cui vuole rendere disponibile il suo microservizio al momento del caricamento dell'interfaccia. (desider)
		3.4.1 L'utente deve avere la possibilità di settare una policy di vendita a consumo di traffico. (des)
		3.4.2 L'utente deve avere la possibilità di settare una policy di vendita a scadenza. (des)
		3.4.3 L'utente deve avere la possibilità di settare una policy di vendita a numero di accessi. (des)
	3.5 APIMarket deve permettere di impostare un testo introduttivo che descriva il suo microservizio al momento del caricamento della sua interfaccia. (obbl)
	3.6 APIMarket deve permettere di impostare dei tag, che descrivano l'utilità del suo microservizio. (fac)
	3.7 APIMarket, al momento della conferma dell'utente, deve mostrare un messaggio che descriva il successo o il fallimento dell'operazione. (obbl)
	3.8 APIMarket deve permettere di caricare la documentazione relativa al microservizio.
4.APIMarket deve fornire una pagina riservata all'utente registrato in cui amministrare il suo account, i suoi messaggi e le interfacce che ha pubblicato. (obbl)
	4.1 APIMarket deve permettere all'utente di modificare i dati del suo profilo. (fac)
		4.1.1 APIMarket deve permettere all'utente di modificare il nome e il cognome del suo profilo, seguendo gli stessi standard qualitativi della registrazione.
		4.1.2 APIMarket deve permettere all'utente di modificare la password del suo profilo, seguendo gli stessi standard qualitativi della registrazione.
		4.1.3 APIMarket deve permettere all'utente di modificare l'indirizzo email collegato al suo profilo, seguendo gli stessi standard qualitativi della registrazione.
		4.1.4 APIMarket deve permettere all'utente di modificare il numero di telefono collegato al suo profilo, seguendo gli stessi standard qualitativi della registrazione.
		4.1.5 APIMarket deve permettere all'utente di modificare il suo paese di provenienza, seguendo gli stessi standard qualitativi della registrazione.
		4.1.6 APIMarket deve permettere all'utente di modificare i dati della sua carta di credito, o di inserirne una nel caso non ne abbia mai inserita una.
		4.1.7 APIMarket deve mostrare all'utente un'errore qualora egli modifichi i suoi dati senza rispettare gli standard minimi.
	4.2 APIMarket deve permettere all'utente di visionare tutti i messaggi che ha scritto, catalogati per data di scrittura. (des)
	4.3 APIMarket deve fornire una sezione notifiche che segnali all'utente modifiche ai prodotti da lui acquistati. (fac)
		4.3.1 APIMarket deve dare la possibilità all'utente di leggere, bloccare e cancellare le notifiche.
	4.4 APIMarket deve fornire all'utente una sezione nella sua area privata in cui supervisionare e modificare le interfacce di microservizi che ha pubblicato. (obbl)
		4.4.1 APIMarket deve fornire all'utente la possibilità di modificare i dati di un microservizio da lui pubblicato.(fac)
			4.4.1.1 APIMarket deve permettere all'utente registrato che ha caricato un'interfaccia di modificarne il nome.
			4.4.1.2 APIMarket deve permettere all'utente registrato che ha caricato l'interfaccia di un microservizio di modificare il prezzo del suo microservizio.
			4.4.1.3 APIMarket deve permettere all'utente registrato che ha registrato l'interfaccia di un microservizio di modificare i tipi di policy di vendita.
			4.4.1.4 APIMarket deve permettere all'utente registrato che ha caricato l'interfaccia di un suo microservizio di modificare il testo introduttivo che descrive il suo microservizio.
			4.4.1.5 APIMarket deve permettere all'utente registrato che ha caricato l'interfaccia di un microservizio di modificare i tag di ricerca associati.
			4.4.1.6 APIMarket deve permettere all'utente registrato che ha caricato l'interfaccia di un microservizio di eliminare o ricaricare la documentazione relativa.
			4.4.1.7 APIMarket deve mostrare un errore all'utente qualora tenti di modificare un campo senza rispettare i requisiti minimi.
		4.4.2 APIMarket deve fornire all'utente la possibilità di eliminare l'interfaccia di un microservizio da lui pubblicata dal sistema. (des)
			4.4.2.1 APIMarket inviare una notifica a tutti i possessori di APIKey riguardanti il microservizio rimosso, avvisandoli che il microservizio è stato rimosso da APIMarket. 
		4.4.3 APIMarket deve fornire all'utente un'area, per ogni microservizio da lui pubblicato, in cui monitorare le sue statistiche. (obbl)
			4.4.3.1 APIMarket deve mostrare al proprietario di un microservizio il numero totale di APIKey vendute.
			4.4.3.2 APIMarket deve mostrare al proprietario di un microservizio il guadagno totale.
			4.4.3.3 APIMarket deve mostrare al proprietario di un microservizio il guadagno mensile.
			4.4.3.4 APIMarket deve mostrare al proprietario di un microservizio l'elenco degli utenti che possiedono una sua APIkey.
			4.4.3.5 APIMarket deve mostrare al proprietario di un microservizio la quantità totale di traffico scambiato.
			4.4.3.6 APIMarket deve mostrare al proprietario di un microservizio l'indice di valutazione degli utenti.
			4.4.3.7 APIMarket deve mostrare al proprietario di un microservizio l'elenco dei commenti degli utenti.
			4.4.3.8 APIMarket deve mostrare al proprietario di un microservizio l'elenco dei crash che ha segnalato il suo microservizio all APIGateway.
			4.4.3.9 APIMarket deve mostrare al proprietario di un microservizio il tempo medio di risposta del microservizio.
			4.4.3.10 APIMarket deve mostrare al proprietario di un microservizio il numero medio di dati scambiati mensile.
			4.4.3.11 APIMarket deve mostrare al proprietario di un microservizio il numero medio di chiamate mensile.
	4.5 APIMarket deve mostrare nella pagina utente il credito di quell'utente in dollari o euro. (obbl)
	4.6 APIMarket deve mostrare nella pagina utente in numero di APICredits in suo possesso. (des)
	4.7 APIMarket deve fornire all'utente la possibilità di eliminare il suo account da APIMarket.
		4.7.1 APIMarket deve chiedere conferma all'utente sulla sua cancellazione dell'account.
		4.7.2 APIMarket deve bloccare l'accesso all'account che si vuole cancellare.
		4.7.3 APIMarket deve inviare una email all'indirizzo email collegato all'account da eliminare in cui viene inviato un link per l'annullamento della procedura, da visitare entro un mese dall'invio dell'email.
			4.7.3.1 Se il link viene visitato entro un mese dalla richiesta allora APIMarket deve annullare la richiesta di cancellazione.
			4.7.3.1 Se il link viene visitato dopo un mese dalla richiesta allora APIMarket deve mostrare un messaggio che comunichi che l'account è stato cancellato.
		4.7.4 APIMarket deve mantere l'account attivo per un mese dalla richiesta di cancellazione, in seguito deve eliminarlo.
			4.7.4.1 APIMarket eliminando un account deve eliminare anche tutte le interfacce di microservizi pubblicate dal possessore di quell'account.
5. APIMarket deve permettere a qualsiasi utente, anche non registrato, di cercare dei microservizi sul sito.
	5.1 APIMarket deve fornire una barra di ricerca per la ricerca dei microservizi caricati.
	5.2 APIMarket deve fornire una ricerca per tag dei microservizi caricati.
	5.3 APIMarket deve permettere di ordinare i risultati della ricerca.
		5.3.1 APIMarket deve permettere di ordinare i risutati per pertinenza.
		5.3.2 APIMarket deve permettere di ordinare i risultati per numero di APIKey vendute.
		5.3.3 APIMarket deve permettere di ordinare i risultati per valutazione degli utenti.
		5.3.4 APIMarket deve permettere di ordinare i risultati per data di caricamento.
6. APIMarket deve permettere a qualsiasi utente, anche non registrato, di visionare la pagina di un microservizio cercato.(obbl)
	6.1 APIMarket deve mostrare il nome del microservizio, la descrizione e il prezzo.(obbl)
	6.2 APIMarket deve dare la possibilità al visitatore di visionare la documentazione relativa. (obbl)
	6.3 APIMarket deve dare la possibilità al visitatore di visionare le statistiche sul microservizio. (obbl)
		6.3.1 APIMarket deve mostrare un numero approssimativo di APIKey venute.
		6.3.2 APIMarket deve mostrare il tempo medio di risposta del microservizio.
		6.3.3 APIMarket deve mostrare il numero medio di dati scambiati mensile.
		6.3.4 APIMarket deve mostrare il numero medio di chiamate mensile.
		6.3.5 APIMarket deve mostrare un livello di attendibilità basato sul numero di crash mensili.
	6.4 APIMarket deve mostrare il nome utente dell'utente che ha pubblicato questa interfaccia di microservizio. (obbl)
	6.5 APIMarket deve dare la possibilità agli utenti registrati e loggati di acquistare una APIKey per il microservizio visitato, utilizzando la carta di credito registrata.(obbl)
		6.5.1 APIMarket deve invitare a registrare una carta qualora si tenti di acquistare una APIKey senza averne registrata una.
		6.5.2 APIMarket deve mostrare un messaggio di conferma per l'acquisto dell'APIKey.
		6.5.3 APIMarket deve mostrare un messaggio che descriva il risultato della transazione.
	6.6 APIMarket deve mostrare la valutazione media data dagli utenti su questo microservizio.
	6.7 APIMarket deve mostrare gli ultimi commenti scritti dagli utenti sul microservizio.
7. APIMarket deve associare una APIKey d'uso per ogni API registrata e per ogni utente che decide di acquistarla. (obbl)
	7.1 Ciascuna APIKey deve essere unica e può essere usata dall'utente che l'ha acquistata per accedere alla API a cui è collegata. (obbl)
		7.1.1 Se la policy di acquisto di una APIKey termina e non viene rinnovata la APIKey cessa di funzionare e non permette più l'utilizzo dell'API a cui è collegata. (obbl)
	7.2 Se la APIKey è contraffatta o non valida allora la chiamata all'API viene bloccata dall'APIMaket. (obbl)
8. APIMarket deve poter rendere visitabile la le pagine utente.
	8.1 APIMarket nella pagina utente deve rendere visibile email, commenti scritti e interfacce di microservizi pubblicate.
	8.2 APIMarket nella pagina utente deve mostrare un indice di affidabilità dell'utente calcolato sulle statistiche dei microservizi da lui pubblicati.
9. APIMarket deve fornire al committente del capitolato un account Admin con poteri di Super-User. (obbl)
    9.1 Un utente Admin deve poter eseguire qualuque funzione che può eseguire un Utente registrato. (obbl)
    9.2 Un utente Admin deve avere la possibilità di modificare i dati anagrafici di qualsiasi utente. (facolt)
    9.3 Un utente Admin deve poter eliminare un commento di qualsiasi utente (facolt)
    9.4 Un utente Admin deve poter eliminare qualsiasi utente. (obbl)
    9.5 Un utente Admin deve poter modificare i dati di una API la cui interfaccia è stata caricata su APIMarket. (obbl)
    9.6 Un utente Admin deve avere la possibilità di modificare la documentazione relativa ad una API sull'APIMarket. (obbl)
    9.7 Un utente Admin deve poter elimindare una API da APIMarket (obbl)
    9.8 Un utente Admin deve poter ricercare qualunque utente tramite una barra di ricerca dedicata (facolt)
